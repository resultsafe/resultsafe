# 📑 Commit Message Convention
# 
# Формат:
# <type>(<scope>): <emoji> <short summary>
# [optional body]
# [optional footer: BREAKING CHANGE: ...]
# 
# type → тип коммита (см. таблицу ниже)
# scope → область (например: domain, application, infrastructure, cli, gui)
# emoji → визуальный маркер
# summary → краткое описание на русском/английском
# 
# --- Типы коммитов
# 
# | Type     | Emoji | Описание                                                    |
# |----------|-------|-------------------------------------------------------------|
# | feat     | ✨    | Добавление новой фичи                                       |
# | fix      | 🐛    | Исправление бага                                            |
# | docs     | 📚    | Изменения в документации                                    |
# | style    | 🎨    | Правки форматирования (без изменения логики)                |
# | refactor | 🔧    | Рефакторинг кода без изменения поведения                    |
# | test     | 🧪    | Добавление или изменение тестов                             |
# | chore    | 🧹    | Обслуживание проекта (зависимости, скрипты, конфиги)        |
# | ci       | ⚙️    | Изменения в CI/CD                                           |
# | build    | 🏗️    | Изменения, влияющие на сборку (Cargo, Webpack, Vite и т.д.) |
# | perf     | 🚀    | Улучшения производительности                                |
# | revert   | ⏪    | Откат предыдущего коммита                                   |
# | security | 🔒     | Изменения, связанные с безопасностью                       |
# | deps     | 📦     | Обновление зависимостей                                    |
# | env      | 🌍     | Изменения окружения (.env, Docker, Devcontainer)           |
# | ux/ui    | 💅     | Изменения в UX/UI (если используется GUI)                  |
