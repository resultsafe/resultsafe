# Changeset Template

Use this structure for all release notes:

```md
---
"@resultsafe/core-fp-result": patch
---

Short user-visible summary.

Release notes:
- Added:
- Changed:
- Fixed:
```

SemVer guidance:
- `patch` - backward-compatible fixes and docs/internal improvements.
- `minor` - backward-compatible API additions.
- `major` - breaking API/type behavior changes.
